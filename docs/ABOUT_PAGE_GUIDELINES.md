# 关于页面与更新区域推荐规范

本规范适用于引用 `DesktopUpdateKit` 的 Windows 单 EXE 应用。它统一用户在不同小程序中的更新体验，但不接管各项目的完整“关于”页面。

## 1. 职责边界

共享组件负责：

- 检查版本和读取 `update.json`。
- 读取更新说明、版本号、大小和校验值。
- 下载、暂停、继续、取消、节点切换和进度状态。
- SHA-256 校验、释放 `UpdaterStub` 和启动安装。
- 提供稳定的更新状态和用户操作接口。

宿主项目负责：

- “关于”页面的窗口、页面或选项卡布局。
- 应用名称、图标、简介、开发者和项目历史。
- GitHub、反馈、许可证和隐私链接。
- 本项目的主题、字体、语言和其他业务说明。
- 将共享更新状态绑定到页面控件。

共享组件不得把开发者名称、GitHub 仓库、应用介绍或页面布局写死在更新核心库中。

## 2. 推荐页面结构

页面推荐按以下顺序组织：

```text
关于
├─ 应用名称、图标和简介
├─ 更新区域
│  ├─ 当前版本
│  ├─ 检查更新
│  ├─ 新版本信息和更新说明
│  ├─ 下载控制与进度
│  └─ 立即安装
├─ 开发者信息
├─ GitHub 项目与更新历史
└─ 项目自定义信息
```

更新区域建议放在页面上方。用户不应为了检查更新进入二级菜单，也不应依赖系统弹窗完成普通更新操作。

## 3. 固定交互规则

所有接入项目应保持以下语义：

| 状态 | 页面行为 |
| --- | --- |
| 初始状态 | 显示当前版本和“检查更新”按钮 |
| 检查中 | 禁用重复检查，显示应用内状态 |
| 已是最新 | 页面内显示结果，不弹系统错误框 |
| 发现更新 | 显示目标版本、更新说明和“下载更新” |
| 下载中 | 显示进度、速度、节点、暂停/继续、取消 |
| 后台下载 | 页面关闭后任务继续，不自动退出主程序 |
| 下载完成 | 显示校验完成和“立即安装” |
| 下载失败 | 显示可读错误和重新下载入口 |
| 安装启动 | 明确提示主程序即将退出并交给更新助手 |

“使用加速节点”开关和“切换加速节点”按钮只在发现可用更新后显示。关闭页面默认暂停下载；用户明确选择后台下载后才继续运行。

下载器不得在检查失败、节点失败、SHA-256 不匹配或安装失败时调用宿主项目的 Windows 弹窗。宿主可以自行提供辅助入口，但默认错误应显示在页面内。

## 4. 可定制范围

项目可以定制：

- 页面整体风格、颜色、图标和布局容器。
- 应用简介、开发者名称、历史内容和链接。
- 更新区域周围的项目说明。
- 文案翻译和无障碍辅助文本。

项目不应改变：

- “检查更新 → 下载并校验 → 明确安装”的顺序。
- 下载完成前不得退出主程序。
- SHA-256 校验失败不得进入安装流程。
- UI 不得自行实现 Range 下载、分块合并、节点特殊逻辑或替换回滚。

如果多个项目使用相同的 WPF 技术栈，可以在未来增加可选的 `DesktopUpdateKit.Wpf` 更新面板。该面板只能提供默认外观和绑定接口，不能替代宿主项目的“关于”页面，也不得让核心 `DesktopUpdateKit` 依赖 WPF。

## 5. 元数据和 JSON 边界

`release.config.json` 可以包含本地的可选 `about` 元数据，例如：

```json
{
  "about": {
    "displayName": "超级中键",
    "developer": "wet86y",
    "historyUrl": "https://github.com/wet86y/clipboard-wheel"
  }
}
```

这部分用于本地项目约束、未来页面生成或发布检查，不会自动写入远程 `update.json`。

远程 `update.json` 只描述版本和发布资产。它不得携带 XAML、脚本、任意 HTML、远程图片或未经约束的 UI 配置。更新说明按纯文本或受限 Markdown 处理，不能成为更新程序执行代码的入口。

## 6. 版本和链接

- 当前版本显示应来自宿主程序集/项目版本源，不应由页面手工填写。
- 目标版本、更新说明和资产校验值来自已校验的 `update.json`。
- GitHub 历史链接属于宿主项目元数据，不由下载核心推断。
- 下载地址仍由共享组件根据仓库、固定 Tag 和资产名构造；远程节点只能包装该官方地址。

## 7. 验收要求

新项目接入时至少验证：

- 中文和带空格路径下关于页可正常显示和下载。
- 检查失败、下载失败、暂停、取消和 SHA-256 失败均只显示应用内状态。
- 关闭页面后的默认行为是暂停，而非丢失任务或自动安装。
- 下载完成后必须由用户明确点击安装。
- 当前项目的开发者、历史链接和版本号没有被共享组件覆盖。
